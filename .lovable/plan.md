# План: rclone в инструкции — поднять в приоритет и уточнить лимит 300 МБ

## Проблема

В текущей версии `docs/self-hosted-basemap-setup.md` Шаг 5 предлагает
два варианта заливки: **A. через веб-UI** (как «проще») и **B. через
rclone** (как «быстрее для шрифтов»).

Но `georgia.pmtiles` = 373 МБ, а Cloudflare R2 web-UI режет одиночный
аплоад на **300 МБ**. Поэтому Вариант A пользователю не подходит —
нужно сразу идти на rclone.

## Что меняю

Один файл: **`docs/self-hosted-basemap-setup.md`**, только Шаг 5.

1. **Перевернуть порядок вариантов**: rclone становится **основным**
   (Вариант A), веб-UI — опциональным fallback-ом (Вариант B) для
   тех, кто хочет залить только мелкие папки `fonts/`/`sprites/`
   и обновить большой файл позже.

2. **В начале Шага 5 добавить явный warning**:
   > ⚠️ `georgia.pmtiles` — 373 МБ. Веб-интерфейс R2 не принимает
   > файлы больше **300 МБ** (это лимит Cloudflare). Поэтому большой
   > файл заливаем через `rclone` или `wrangler` — это единственный
   > рабочий путь для самого pmtiles. Мелкие папки `fonts/` и
   > `sprites/` тем же `rclone` зальются за секунды.

3. **Уточнить инструкцию rclone**:
   - явно сказать, где взять **Account ID** (R2 dashboard →
     правый сайдбар → **Account details**, не в Overview);
   - дать **endpoint** в актуальном формате:
     `https://<account-id>.r2.cloudflarestorage.com`;
   - в `rclone config` указать конкретные ответы для **всех** полей,
     которые он спросит (включая `acl` → `private`, `no_check_bucket`
     → `false`, и т.д.), чтобы не было пауз с непонятными вопросами;
   - добавить пример `rclone copy` с флагом `--s3-upload-cutoff 200M
     --s3-chunk-size 100M` — это включает multipart upload и позволяет
     перезалить, если оборвалось соединение;
   - после заливки `pmtiles` — команда проверки:
     `rclone size r2:metrics-basemap` (должно показать ~383 МБ).

4. **Веб-UI оставить как Вариант B** с явным «работает только для
   `fonts/` и `sprites/`, не для `georgia.pmtiles`».

## Что НЕ трогаем

- Код (`src/lib/map-style.ts`, компоненты карты) — не зависит от способа
  заливки.
- Шаги 1–4 и 6–8 инструкции.
- CORS-блок Шага 4 — он только что починен.
