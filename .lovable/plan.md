# Привязка карты Тифлиса 1898 в QGIS и подключение в проект

Текущая привязка по 4 углам через `image`-source даёт большую погрешность: лист бумаги деформирован, проекция оригинала (вёрстка от меридиана Ферро + многогранная проекция Российского ВТУ) не совпадает с EPSG:3857. Решение — георефенс по 15–25 опорным точкам с тонкопластинчатой трансформацией (Thin Plate Spline), затем нарезка в XYZ-тайлы и трассировка полигонов участков.

## Этап 0. Что вам понадобится

- QGIS 3.34+ LTR (бесплатно, qgis.org/download)
- Исходный скан карты (тот, что вы уже прислали через Google Drive — у меня лежит как высокое разрешение, могу отдать обратно при необходимости)
- 1–2 часа времени на первую привязку

## Этап 1. Подготовка проекта QGIS

1. Открыть QGIS → **Project → New**.
2. В правом нижнем углу нажать на CRS (по умолчанию EPSG:4326) → выбрать **EPSG:3857 — WGS 84 / Pseudo-Mercator**. Это критично: тайлы для MapLibre должны быть в 3857.
3. Подложить современную карту как референс:
   - Browser panel → **XYZ Tiles** → правой кнопкой → **New Connection**
   - Name: `OSM`, URL: `https://tile.openstreetmap.org/{z}/{x}/{y}.png`
   - Перетащить `OSM` на холст
4. Дополнительно полезно подключить спутник (для русел рек и старых улиц): `https://server.arcgisonline.com/ArcGIS/rest/services/World_Imagery/MapServer/tile/{z}/{y}/{x}`

## Этап 2. Георефенс (привязка скана)

1. Меню **Raster → Georeferencer** (встроен в QGIS 3.x, плагин ставить не надо).
2. В окне Georeferencer: **Open Raster** → выбрать скан карты 1898.
3. **Transformation Settings** (кнопка с шестерёнкой):
   - Transformation type: **Thin Plate Spline** (TPS) — лучше всего «подтягивает» искажения бумаги
   - Resampling: **Cubic**
   - Target CRS: **EPSG:3857**
   - Output raster: `tbilisi-1898-georef.tif`
   - Compression: **LZW**
   - ✅ Use 0 for transparency when needed
   - ✅ Save GCP points
   - ✅ Load in QGIS when done

### Опорные точки (GCP) — ключевой момент

Нажать **Add Point**, кликнуть в характерное место на старой карте → откроется диалог → нажать **From Map Canvas** → кликнуть в то же место на OSM. Повторить 15–25 раз. Точки распределяйте равномерно по всему листу, не кучкуйте в центре.

4. **File → Start Georeferencing** (зелёная стрелка). На выходе — `tbilisi-1898-georef.tif`, автоматически загрузится поверх OSM. Покрутите прозрачность слоя (Properties → Transparency → 50%) и проверьте совпадение мостов/церквей.

## Этап 3. Нарезка в XYZ-тайлы

1. **Processing Toolbox** (Ctrl+Alt+T) → найти **Generate XYZ tiles (Directory)**.
2. Параметры:
   - Extent: **Use Layer Extent → tbilisi-1898-georef**
   - Minimum zoom: **12**
   - Maximum zoom: **17** (18 даст ×4 размер при малом выигрыше)
   - Tile format: **PNG**
   - DPI: 96
   - Background color: прозрачный (alpha = 0)
   - Tile width/height: 256
   - Output directory: любая папка, например `~/tbilisi-1898-tiles`
3. Run. Займёт 3–10 минут. На выходе структура `{z}/{x}/{y}.png`, итоговый размер обычно 20–80 МБ.

> Если получится >100 МБ — поднимите minzoom до 13 или используйте JPG (но потеряете прозрачность за рамкой листа).

## Этап 4. Трассировка полигонов участков

1. **Layer → Create Layer → New GeoPackage Layer** (или Shapefile):
   - Geometry: **MultiPolygon**
   - CRS: **EPSG:4326** (для GeoJSON удобнее)
   - Fields:
     - `name_latin` (text) — как написано на карте латиницей
     - `name_ru` (text)
     - `name_en` (text)
     - `name_ka` (text)
2. Выделить слой → **Toggle Editing** (карандаш) → **Add Polygon Feature**.
3. Обвести каждый полицейский участок по границам со старой карты (обычно границы — жирные пунктирные линии, идущие по улицам/рекам). По завершении полигона — правый клик, ввести атрибуты.
4. Включить **Snapping** (магнит в тулбаре) на 10 px по vertex+segment, чтобы соседние участки не оставляли щелей. Использовать инструмент **Add Polygon** + **Avoid overlap** или сразу **Digitize with segment**.
5. После всех участков: **Save Edits** → правый клик по слою → **Export → Save Features As**:
   - Format: **GeoJSON**
   - CRS: **EPSG:4326**
   - Coordinate precision: **6**
   - File: `tbilisi-1898-districts.geojson`

## Этап 5. Передать мне результаты

Залейте в чат (или Google Drive):
1. Папку с тайлами `tbilisi-1898-tiles/` (можно zip)
2. Файл `tbilisi-1898-districts.geojson`

## Что я сделаю после получения файлов

1. Положу тайлы в `public/tiles/tbilisi-1898/{z}/{x}/{y}.png`.
2. В `src/lib/tbilisi-historical.ts` переключу `TBILISI_1898` с `kind: "image"` на `kind: "tiles"`:
   ```ts
   export const TBILISI_1898: HistoricalConfig = {
     kind: "tiles",
     tiles: "/tiles/tbilisi-1898/{z}/{x}/{y}.png",
     minzoom: 12,
     maxzoom: 17,
     attribution: "План г. Тифлиса, 1898 · Национальный архив Грузии",
   };
   ```
3. Положу `public/data/tbilisi-1898-districts.geojson` (заменит текущий placeholder с пустым `features: []`).
4. В `TbilisiMap.tsx` слой полигонов уже подключён к этому URL — он автоматически отрисуется: полупрозрачная заливка + контур + латинские подписи по центроидам, под слоем точек церквей.
5. Проверю в превью совпадение мостов и видимость точек поверх старой карты.

## Если что-то пойдёт не так

- **TPS «съезжает» по краям листа** — это нормально, на полях карты обычно нет важных объектов; убедитесь, что GCP равномерно покрывают именно интересующую территорию.
- **Тайлы получаются «лесенкой»** — увеличьте maxzoom до 18, либо в Layer Properties → Symbology включите Resampling: Bilinear перед нарезкой.
- **QGIS падает на TPS с 30+ точек** — попробуйте **Polynomial 2** как компромисс: чуть менее точно, гораздо стабильнее.
- **Не хватает времени на 25 точек** — минимум для приемлемого результата 12 точек, равномерно по листу, с обязательным захватом 4 углов исторического центра.

## Альтернатива на крайний случай

Если QGIS не пойдёт совсем — пришлите хотя бы CSV из 8–12 пар «пиксель скана → lon/lat», и я смогу сделать преобразование сам через `gdal_translate` + `gdalwarp` на своей стороне (но без визуального контроля качество хуже, чем у вас в QGIS с подложкой OSM).
