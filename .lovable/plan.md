## Цель
Карточка точки должна помещаться в любой экран (особенно мобильный), кнопка закрытия — всегда доступна, даже если в точке сотни церквей.

## Что меняем

### 1. Откатываем дробление Тбилиси
В `scripts/build-geojson.ts` убираем блок «split features with very large church lists» (добавленный в прошлой итерации) и пересобираем `public/data/parishes.geojson` — Тбилиси снова станет одной точкой со всеми 149 церквями.

### 2. Перестраиваем карточку точки в `src/components/map/MapView.tsx`
Сейчас карточка — один блок с `p-4`, она растёт по содержимому и вылезает за экран. Делаем её flex-контейнером с ограничением по высоте и внутренней прокруткой:

- Контейнер карточки: `max-h-[min(70vh,560px)]` на мобильном, `max-h-[70vh]` на десктопе; `flex flex-col`; внешний `p-4` убираем.
- **Sticky-шапка** (не скроллится): название селения, подзаголовок-церковь (если короткая) и кнопка `×`. Padding `px-4 pt-4 pb-2`, `bg-card/98 backdrop-blur`, `border-b border-border`. Кнопка закрытия всегда видна сверху.
- **Скроллируемое тело**: `flex-1 overflow-y-auto px-4 py-3 overscroll-contain`. Внутри — текущий `<dl>` с регионом/уездом/годами/пропусками. Длинный список церквей выносим из подзаголовка в отдельную секцию внутри тела (см. ниже), чтобы шапка оставалась компактной.
- **Sticky-футер**: кнопка «Показать радиус 50 км» + строка «рядом N точек». `px-4 py-3 border-t border-border bg-card/98`. Тоже всегда виден.

### 3. Список церквей — отдельная секция со своим скроллом
Если в `properties.church[lang]` больше ~3 элементов (разделитель `|`), показываем их не подзаголовком, а блоком в теле карточки:
- Заголовок секции «Церкви (N)» (i18n: `T.churches`, добавить в `src/lib/i18n.ts`).
- Список `<ul>` с `max-h-48 overflow-y-auto` внутри уже скроллящегося тела — даёт компактное превью + возможность пролистать всё, не растягивая карточку.
- Если 1–3 церкви — оставляем как сейчас (курсив под названием), без отдельной секции.

### 4. Мелочи
- Сохраняем `pointer-events-auto`, `rounded-2xl`, `shadow-2xl`, `backdrop-blur` — визуально карточка не меняется.
- На мобильном карточка по-прежнему `bottom-3 left-3 w-[min(92vw,360px)]`; мобильная горизонтальная легенда уже скрывается при `selected` — оставляем.
- `overscroll-contain` на скроллящемся теле, чтобы свайп внутри списка не двигал карту.

## Технические детали

**Файлы:**
- `scripts/build-geojson.ts` — удалить добавленный split-блок (строки ~189–235), вернуть прямую запись `features`.
- запустить `bun scripts/build-geojson.ts` для регенерации `public/data/parishes.geojson` и `stats.json`.
- `src/components/map/MapView.tsx` — переписать JSX блока `{selected && sel && (...)}` (строки ~370–425): три секции (sticky header / scrollable body / sticky footer), новая секция «Церкви» при `count > 3`.
- `src/lib/i18n.ts` — добавить ключ `churches` (RU: «Церкви», EN: "Churches").

**Поведение скролла:**
```text
┌─────────────────────────────┐
│ Тбилиси              [×]    │ ← sticky header
├─────────────────────────────┤
│ Регион   Тбилисский район   │
│ Годы     1796–1924 (129)    │ ← scrollable body
│ Пропуски Нет пропусков      │
│                             │
│ Церкви (149)                │
│ ┌─────────────────────────┐ │
│ │ Анчисхати Собор         │ │ ← вложенный скролл
│ │ Архангела               │ │   max-h-48
│ │ ...                     │ │
│ └─────────────────────────┘ │
├─────────────────────────────┤
│ [📍 Показать радиус 50 км]  │ ← sticky footer

```

**Чего НЕ трогаем:** карта, источники/слои MapLibre, фильтры по периодам, поиск, легенды, статистика, embed-режим, OG-теги.

## Проверка
- На мобильном (375×667) карточка Тбилиси помещается, `×` виден сверху, кнопка радиуса — снизу, список церквей листается внутри.
- На обычной точке с 1–2 церквями карточка выглядит как раньше (короткая, без секции «Церкви»).
- Карта Тбилиси снова показывает одну точку без суффиксов `(1/4)`.
