## Почему так происходит

В `src/routes/admin.tsx` для каждого предложения координат рендерится **iframe с `https://www.openstreetmap.org/export/embed.html`**, а рядом — ссылка `OSM` с `target="_blank"`.

OpenStreetMap отдаёт заголовки (`X-Frame-Options` / CSP `frame-ancestors`), которые запрещают встраивание страниц openstreetmap.org в чужие iframe. Поэтому:

1. Сам встроенный мини-предпросмотр карты в админке у вас уже не грузится — браузер блокирует его с `ERR_BLOCKED_BY_RESPONSE` и показывает текст «www.openstreetmap.org is blocked».
2. Превью Lovable открывает админку внутри собственного `<iframe sandbox=...>`. В этом sandbox по умолчанию нет `allow-popups`, поэтому клик по ссылке `<a target="_blank">` фактически открывается **внутри текущего iframe**, а не в новой вкладке. И снова ловит тот же запрет фрейминга от OSM.

То есть проблема не в данных и не в координатах, а в том, что мы пытаемся показать OSM внутри iframe.

## Что предлагаю сделать

Полностью убрать зависимость от страниц openstreetmap.org в админке.

1. **Заменить iframe-предпросмотр** (строки ~502–538 в `src/routes/admin.tsx`) на собственный мини-MapLibre-предпросмотр, использующий ту же стилизацию тайлов, что и основная карта проекта (`src/lib/map-style.ts`). Один маркер по `r.lat`, `r.lon`, зум `r.zoom`, статичный (без drag/zoom) — этого достаточно, чтобы оценить точку. Тайлы будут грузиться с того же источника, что и на `/map`, никаких ограничений по фреймингу.

2. **Ссылка «OSM»** (строки 428–435 и 527–534): оставить URL тот же, но открывать не через `<a target="_blank">`, а через `onClick={() => window.open(href, '_blank', 'noopener,noreferrer')}`. В опубликованной версии (`metrics.datatells.info`) это поведёт себя как обычная новая вкладка; внутри Lovable-превью, где `window.open` тоже блокируется sandbox'ом, в качестве fallback показывать `toast` с подсказкой «Откройте админку в новой вкладке, чтобы перейти на OSM» и копировать ссылку в буфер. Для удобства добавить рядом маленькую кнопку «Скопировать ссылку».

3. Тексты («Откройте на OSM», fallback-сообщение) — на русском, в стиле остальных строк админки.

Никакой бизнес-логики (одобрение/отклонение предложений, схемы БД) не трогаем — только два визуальных блока в `src/routes/admin.tsx`.

- Удалить переменные `bbox` и `embedSrc`, оставить только `osmHref`.
- Хелпер `openExternal(href)`:
  ```ts
  const w = window.open(href, '_blank', 'noopener,noreferrer');
  if (!w) {
    await navigator.clipboard.writeText(href).catch(() => {});
    toast.info('Ссылка скопирована — откройте её в новой вкладке');
  }
  ```
  использовать и в верхней ссылке (строка 428) и в нижней (527).
