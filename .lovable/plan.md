## Идея

В админке появляется новая вкладка «AI-геокодер» — она берёт сёла из `unlocated.json` (5198 групп) и для каждого пытается найти координаты через внешний геокодер. Кандидат принимается только если попадает в правильный регион/уезд. Найденные точки попадают в очередь модерации, где админ за один клик публикует их как `feature_overrides` (action `add`) — то же самое, что сейчас делает форма «Добавить координаты» вручную.

## Почему именно так

- Полностью автоматическая запись опасна: ИИ путает одноимённые сёла (Носте, Чала, Земо/Квемо…), а у нас уже были случаи с неверным регионом. Поэтому — **очередь предложений с модерацией**, а не прямой коммит.
- Уже существует вся инфраструктура: `feature_overrides` с `action='add'`, RLS под админа, история правок, кнопки публикации. Агент просто наполняет данные, дальше работает существующий пайплайн.

## Выбор геокодера

Google Maps Geocoding API — платный (нужен ключ + биллинг) и его ToS запрещает хранить координаты без отображения на Google-карте. У нас MapLibre + OSM, поэтому Google юридически не подходит.

Рекомендую один из:
1. **Nominatim (OSM)** — бесплатно, structured query (`village=…`, `state=…`, `country=Georgia`), лимит 1 req/sec. Идеально для нашего объёма (5198 × 1с ≈ 1.5 часа на проход).
2. **LocationIQ / Geoapify** — те же данные OSM, выше лимиты, бесплатный тариф 5k/день.
3. **Lovable AI (Gemini)** как «второй мозг» поверх кандидатов: получает топ-3 от Nominatim + историческое название/уезд/церковь и выбирает лучший вариант либо отклоняет все.

По умолчанию: Nominatim + Gemini-арбитр. Без новых платных ключей.

## Архитектура

```
[Админка] → кнопка «Запустить AI-геокодер»
   │
   ▼
[server fn: enqueueGeocodingBatch]  — берёт N необработанных сёл из unlocated.json
   │
   ▼
[server fn: geocodeOne]  (по одному, с задержкой)
   ├─ Nominatim structured search (ru/en/ka варианты имени + регион + Georgia)
   ├─ фильтр: точка должна попадать в bbox региона/уезда (или в радиус N км от центра)
   ├─ Gemini-арбитр: «вот село X, уезд Y, церковь Z, кандидаты [...] — какой подходит и насколько уверен (0-1)?»
   └─ запись в новую таблицу geocoding_suggestions
   │
   ▼
[Админка → вкладка «Предложения геокодера»]
   ├─ список с превью на мини-карте, score, источник
   ├─ кнопки: Принять / Отклонить / Править вручную
   └─ «Принять» → INSERT в feature_overrides (action='add', published=true)
```

## Что нужно добавить

### Данные
- Таблица `geocoding_suggestions`: `id, settlement_key, settlement/region/uezd jsonb, lat, lon, source, confidence, candidates jsonb, status (pending/accepted/rejected), reviewed_by, created_at`. RLS — только админы.
- Bbox/центры регионов и уездов Грузии — небольшой статический JSON в `scripts/data/regions-bbox.json` (один раз собирается из существующего GeoJSON).

### Сервер (TanStack server functions, защищённые `requireSupabaseAuth` + проверка роли admin)
- `runGeocoderBatch({ limit, uezd? })` — обрабатывает партию, пишет в `geocoding_suggestions`.
- `acceptSuggestion(id, { lat, lon })` — создаёт `feature_overrides` `add` (как сейчас при ручном вводе из `UnlocatedPanel`) и помечает suggestion `accepted`.
- `rejectSuggestion(id, reason)`.

### UI в админке
- Вкладка «AI-геокодер»:
  - кнопка «Найти координаты для следующих 100 сёл» (с прогресс-баром)
  - фильтр по уезду
- Вкладка «Предложения»:
  - карточка села: историческое имя, годы, уезд, церковь
  - top-3 кандидата от Nominatim, выбранный Gemini, confidence
  - мини-карта (уже есть `AdminMiniMap`)
  - кнопки Принять / Отклонить / Открыть в OSM

### Секреты
- Если выберем Nominatim — секреты не нужны (только User-Agent).
- Если LocationIQ/Geoapify — `add_secret` для API-ключа.
- Lovable AI Gateway уже подключён (`LOVABLE_API_KEY` есть).

## Ограничения и риски

- Нужен ручной апрув — иначе будет мусор (одноимённые сёла, исчезнувшие населённые пункты, ошибки в исторических названиях).
- Nominatim rate limit 1 rps → пакетная обработка в фоне, не блокируем UI.
- Сёла без указания уезда (`uezd=""`) геокодим только если confidence очень высокий (1 кандидат в Грузии).
- ToS: использовать Nominatim/OSM можно, Google Maps Geocoding для хранения координат вне Google-карты — нельзя.

## Этапы

1. Миграция: таблица `geocoding_suggestions` + RLS.
2. Server functions: `runGeocoderBatch`, `acceptSuggestion`, `rejectSuggestion`.
3. Скрипт сбора `regions-bbox.json` из текущего GeoJSON.
4. UI в `/admin`: две вкладки + мини-карта в карточке предложения.
5. Прогон на 50 тестовых сёлах, оценка качества, тюнинг порога confidence.

## Альтернатива «попроще»

Если не хочется новой таблицы и UI — сделать **CLI-скрипт** `scripts/ai-geocode.ts`, который пишет результаты в JSON, а ты глазами просматриваешь и через существующую панель «Сёла без координат» вбиваешь подтверждённые. Быстрее в реализации, но менее удобно для регулярной работы.
