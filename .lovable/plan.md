
## 1. Новый переключатель карт на /tbilisi

### Логика
Единый источник управления — один `<select>` со значениями:
- `none` — «Без старой карты» (исторический растр выключен)
- `1898` — Тифлис, 1898
- `1904` — Тифлис, 1904
- (любая будущая запись из `HISTORICAL_MAPS`, у которой `config.kind === "tiles"`)

Чекбокс «Историческая карта» убираем полностью. Состояние `historicalOn` выводится из `historicalMapId !== "none"`. В URL продолжаем хранить `hm` (id) и `o` (прозрачность); параметр `h` больше не нужен (поддержим обратное чтение для старых ссылок: если `h=0` → `hm="none"`). Чекбокс «Полицейские участки» оставляем отдельной строкой — он не зависит от выбора растра.

### Desktop (≥1024px)
- Панель «Исторический слой» переезжает в **левый нижний угол** (`bottom-4 left-4`), фиксированная ширина ~240px.
- Содержимое: заголовок + `select` карты + слайдер прозрачности (видимый только если выбрана карта) + чекбокс «Полицейские участки».

### Tablet (sm…lg, 640–1023px)
- Та же панель, но компактная: схлопывается в **кнопку-пилюлю** «Слои / 1904» в левом нижнем углу; по клику открывается поповер с тем же содержимым, что и десктопная панель. Так она не перекрывает карту по умолчанию.

### Mobile (<640px)
- Кнопка `Layers` в нижнем ряду действий открывает **bottom sheet** (Drawer) со списком вариантов: `Без старой карты`, `Тифлис, 1898`, `Тифлис, 1904`, слайдером прозрачности и чекбоксом участков. Текущий выбор подсвечен. Тап по варианту = выбор + автозакрытие. Так на мобильном тоже доступны все карты, не только 1898.

### Кнопка «i» (атрибуция)
MapLibre по умолчанию в `compact: true` иногда стартует раскрытым. Принудительно сворачиваем атрибуцию при инициализации (удалением CSS-класса `maplibregl-compact-show` после добавления контрола / либо `attributionControl: false` + `new AttributionControl({compact: true})` и `setTimeout` снять класс). Раскрытие — только по клику. Применяется и на /tbilisi, и в админ-карте.

## 2. Админка → вкладка «Тбилиси» (`TbilisiCoordEditorPanel`)

- Из `<select>` карт показывать **только записи с `config.kind === "tiles"`** (сейчас это 1898 и 1904). Заготовки 1735/1850/1920 скрываем из UI (в `HISTORICAL_MAPS` они остаются — для будущего, но фильтруем по наличию `config`).
- Убираем чекбокс «Старая карта» — `histOn` выводится из факта выбора карты; добавляем пункт `Без подложки` в начале списка.
- Переключатель «Показ: Кроме high / Только low / Все» вместе с «По году карты / Все годы» объединяем в один смысловой блок «Фильтр точек», подписываем явно:
  - **Точки:** `Только проблемные` (= not_high), `Только низкоточные` (= low_only), `Все`.
  - **По дате:** `Только ≤ года карты` ↔ `Все годы`.
  Кнопки помечаем иконками и тултипами; активная — жирно подсвечена.
- При первом открытии вкладки точки сейчас «не появляются» — проверить, не блокирует ли инициализацию `selectedMap?.config` (после фильтрации первой картой будет 1898, у которой `config` есть → точки будут). На всякий случай инициализируем `mapId` дефолтом `"1898"` и `histOn = true` явно; маркеры точек НЕ зависят от `histOn`, чтобы они показывались даже без выбора подложки.

## 3. Тестирование UX/UI

Используем уже подключённый Playwright (`playwright.config.ts`, проекты `mobile-375`, `mobile-390`, `desktop-1280`). Добавляем спеки рядом с `e2e/map-overlays.spec.ts`:

1. **e2e/tbilisi-historical-switcher.spec.ts**
   - На desktop: панель находится в левом нижнем углу, не перекрывает карточку выбранного объекта.
   - Смена `select` `none → 1898 → 1904` обновляет URL (`?hm=...`), слой подложки появляется/исчезает.
   - На tablet: дефолт — закрыт; клик по пилюле открывает поповер; поповер не выходит за границы вьюпорта.
   - На mobile: тап по кнопке `Layers` открывает sheet со всеми 3 опциями; выбор закрывает sheet и обновляет URL.
   - Reload сохраняет выбор (URL/hm).

2. **e2e/tbilisi-attribution.spec.ts** — кнопка «i» в начальном состоянии свёрнута; клик раскрывает; повторный — сворачивает; ничего не торчит под кнопками управления.

3. **e2e/admin-tbilisi-points.spec.ts** (логином через тестовый аккаунт или мок — как сейчас сделано в `integration-tests/admin-*`)
   - После открытия `/admin` → таб «Тбилиси»: точки рендерятся сразу (count > 0).
   - В выпадашке только 1898 и 1904; «нет данных» вариантов нет.
   - Переключение фильтров «Точки» и «По дате» меняет количество видимых маркеров.

4. **Визуальная регрессия (лёгкая):** в тех же спеках делаем `expect(page).toHaveScreenshot()` для трёх вьюпортов на ключевых состояниях (закрыт/открыт переключатель, выбран 1904). Baseline кладём в `e2e/__screenshots__`, в CI запускаем как «non-blocking review» (отдельный workflow или allow-failure), чтобы не блокировать мержи на мелких пиксельных дельтах.

5. **Юнит-тест для логики выбора:** небольшой тест в `src/lib/tbilisi-historical.test.ts` — `HISTORICAL_MAPS.filter(m => m.config?.kind === "tiles")` возвращает ровно `["1898","1904"]` (защита от случайной регистрации заготовки в UI).

Скрипты в `package.json` уже есть для `bunx playwright test`; добавим npm-скрипт `test:e2e:tbilisi` с фильтром по этим спекам для локальной быстрой прогонки.
