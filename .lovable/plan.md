## Цель

Объединить строки CSV с одинаковыми координатами в одну точку на карте, аккумулируя периоды и пропущенные годы из всех связанных церквей/книг этого селения.

## Логика объединения

Группировка идёт по ключу `lat|lon` (округлённые до 6 знаков, чтобы избежать плавающих расхождений).

Для каждой группы:
- **settlement / region / uezd** — берём первое непустое значение (обычно совпадают у всех строк группы).
- **church** — собираем все непустые названия церквей через `; ` (например: `Спасителя; Архангела`). Если все пусты — пусто.
- **yearsRaw** — объединяем годы всех строк в одно множество, потом сжимаем компактно (`1830-1838, 1840, 1851-1870`).
- **missingRaw** — пропуски считаем по объединённому множеству: для каждого года `y` от минимума до максимума диапазона, если `y` не в множестве → это пропуск. Сжимаем тем же компактным форматом.
- **startYear** — минимум среди всех `Начальный год` группы.
- **endYear** — максимум.
- **coverage** — размер объединённого множества лет.
- **missingCount** — длина вычисленных пропусков.
- **bucket** — по объединённому `startYear`.

Это даёт корректный, единый ответ на каждой точке: один период жизни метрических книг селения и реальные пропуски в нём.

## Ожидаемый эффект

- Число точек: с **2603** → **1391** (меньше «дублей» при клике).
- Точки с непустой строкой «Пропущенные годы»: с **575** → **445** (32% от точек), а на остальных будет показано «Нет пропусков», что теперь действительно соответствует данным.
- Поиск в `SearchPanel` будет находить одну запись на селение (для нескольких церквей — с конкатенированным названием).

## Технические детали

Файл `scripts/build-geojson.ts`:

1. Добавить вспомогательную функцию `compactYears(years: number[]): string` — та же логика, что в `src/lib/i18n.ts` (диапазоны через `-`, отдельные года через `, `).
2. Заменить текущий цикл `for (let i = 0; i < len; i++)` на двухпроходный:
   - **Проход 1** — собрать сырые записи (как сейчас) в массив, сразу отбрасывая строки без координат / без `startYear`.
   - **Проход 2** — сгруппировать по ключу `${lat.toFixed(6)}|${lon.toFixed(6)}` в `Map<string, RawRow[]>`.
   - Для каждой группы построить одну `Feature` по правилам выше.
3. Поле `id` — индекс в итоговом массиве features (как сейчас).
4. `stats.json`: `total` теперь = число строк CSV (как было), `withCoords` = число строк с координатами (как было); добавить `uniqueLocations = features.length`, чтобы отразить реальное число точек на карте.
5. Запустить пересборку: `bun scripts/build-geojson.ts`.

В рантайме (`MapView.tsx`, `i18n.ts`, `SearchPanel`) ничего менять не нужно — структура `properties` сохраняется, новый рендер «Пропущенные годы / Нет пропусков» уже корректно обработает оба случая.

## Что вне рамок

- Восстановление 933 строк без координат — нужен геокодинг, отдельная задача.
- Если две разные церкви одного селения геокодированы в одну и ту же точку и пользователь хочет видеть их как отдельные записи — это потребует разнести их по координатам (или добавить отдельный список церквей в карточке). Сейчас они объединяются.
