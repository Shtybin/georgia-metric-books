## Цель

Привести фильтр «Конфессия / община» основной карты к той же системе категорий, что и на карте Тбилиси (`Confession` из `src/lib/i18n-tbilisi.ts`), сделать так чтобы точка с церквями разных конфессий подсвечивалась под каждый соответствующий фильтр, и однократно прогнать датасет `public/data/parishes.geojson` для автоматического присвоения конфессий.

## 1. Единая таксономия категорий

Заменить локальный список из 8 категорий в `src/lib/parishCategory.ts` на 12 типов из карты Тбилиси:

`orthodox_georgian, orthodox_russian, orthodox_military, armenian_apostolic, greek_orthodox, roman_catholic, lutheran, jewish, molokan, baptist, assyrian, other`

- Цвета и порядок берутся из `CONFESSION_COLORS` / `CONFESSION_ORDER` (`i18n-tbilisi.ts`) — без дублирования.
- Подписи в легенде основной карты берутся из `tT(lang).confessionsShort` (короткие: «грузины / армяне / русские / военные …»), всплывающая подсказка — из `confessions` (полные).
- Старые ключи `i18n.ts → categoryName.{georgian,armenian,…}` удаляются, останется один источник истины.

## 2. Мультикатегоризация точки

Точка может содержать несколько церквей в поле `church` (разделители `|`, `;`, `/`, ` – `). Сейчас точке присваивается одна категория `category`. Меняем на массив `categories: Confession[]`.

- Каждый токен церкви категоризируется отдельно; получаем уникальный набор категорий точки.
- Цвет круга по-прежнему один (берётся из `BUCKET_COLORS` по `bucket` — это не меняется, цвет = период метрических книг). Категория управляет только видимостью под фильтром.
- Фильтр-выражение MapLibre переключается с `["in", ["get","category"], ["literal", arr]]` на пересечение массивов: `["any", ...enabled.map(c => ["in", c, ["get","categories"]])]`. Точка с `[armenian_apostolic, orthodox_georgian]` появляется и при выборе «армяне», и при выборе «грузины».

## 3. Логика автоматической классификации (приоритет сверху вниз)

Применяется к каждому токену церкви:

1. **Совпадение с датасетом Тбилиси** (`public/data/tbilisi-churches.json`, 99 церквей с уже выставленным `confession`). Если точка попадает в Тбилиси (по `isInsideTbilisi` или `settlement == "Тбилиси/Тифлис"`), нормализованное имя церкви сопоставляется с именами из tbilisi-churches (по совпадению ключевых слов «Св. X», «Архангелов в Y» и т. д.) — берётся его `confession`.
2. **Явные ключевые слова в названии церкви** (русский / английский / грузинский):
   - `армян|григориан|haykakan|სომხ` → `armenian_apostolic`
   - `католич|римско|catholic|კათოლიკ` → `roman_catholic`
   - `лютеран|кирх|lutheran|kirche|ლუთერ` → `lutheran`
   - `синагог|еврейск|иудей|jewish|synagogue|სინაგოგ` → `jewish`
   - `молокан|молоканск|molokan|მოლოკან` → `molokan`
   - `баптист|baptist|ბაპტისტ` → `baptist`
   - `ассир|айсор|assyrian|aysor|ასირი` → `assyrian`
   - `греческ|греч.|greek|ბერძნ` → `greek_orthodox`
   - `военн|полков|гарнизон|казач|military|regimental|garrison` → `orthodox_military`
   - `единовер|старообряд|русский приход|old believer` → `orthodox_russian`
3. **Эвристика по селению/уезду/региону** (таблица `LOCATION_HINTS` в новом `src/lib/confessionRules.ts`):
   - Армянские очаги: Ахалкалакский уезд (целиком), сёла Ахалцихского уезда (Хертвиси, Хизабавра, Цхалтбила …), Лорийский участок Борчалинского уезда, армянские кварталы в Гори/Сигнахи. → добавляет `armenian_apostolic`.
   - Немецкие колонии (Katharinenfeld/Болниси, Elisabethtal/Асурети, Marienfeld/Сартичала, Александерсхильф, Петерсдорф) → `lutheran`.
   - Молоканские/духоборские сёла (Гореловка, Орловка, Ефремовка, Спасовка, Богдановка, Ново-Михайловка, Воронцовка, Башкичет, Привольное) → `molokan`.
   - Греческие сёла Цалкского/Тетрицкаройского/Дманисского участка (Цалка, Бешташени, Авранло, Триалет, Сантас) → `greek_orthodox`.
   - Еврейские общины (Ахалцихе, Они, Кулаши, Лайлаши, Сачхере, Гори-еврейский квартал) → `jewish`.
   - Старообрядческие/русские крестьянские поселения (Бомборы, Анастасиевка, Свободное, Михайловское) → `orthodox_russian`.
   - Военные приходы при крепостях / штабах (Ананури, Сурам, Кутаис-Цитадель, Хорошани) — если в названии нет других маркеров → `orthodox_military`.
4. **Дефолт**: `orthodox_georgian` (Грузинский экзархат РПЦ — основная масса сельских приходов).

Если точка попадает под пункт 3 (армянский/немецкий/молоканский ареал), но в названии есть «Св. Георгия/Богородицы …» без явных маркеров — категории **складываются**: к ареальной добавляется `orthodox_georgian`, чтобы не потерять православные приходы в смешанных сёлах.

## 4. Аудит всего датасета (одноразовый прогон)

Новый скрипт `scripts/audit-confessions.ts` (`bun run scripts/audit-confessions.ts`):

- Читает `public/data/parishes.geojson` + `public/data/tbilisi-churches.json`.
- Применяет логику п.3 к каждой точке, разворачивая `church` по разделителям.
- Записывает:
  - `public/data/confession-audit.json` — распределение по категориям, число «мультиконфессиональных» точек, топ-50 «подозрительных» (классификация изменилась от ареальной vs названия), список Тбилисских точек, которым удалось/не удалось привязать конфессию из tbilisi-churches.
  - `src/lib/confessionOverrides.generated.ts` — карта `id → Confession[]` для тех 99 тбилисских церквей, чьи конфессии берутся напрямую из выверенного датасета (без эвристик).
- Печатает сводку в stdout (для прикладывания к PR/коммиту).

Скрипт можно перезапускать после добавления новых правил в `LOCATION_HINTS` — `confession-audit.json` обновляется и становится «снимком» текущей классификации.

## 5. Ручные правки (override-таблица)

`src/lib/confessionOverrides.ts` — пустой по умолчанию объект:

```ts
// id точки (или нормализованный ключ "settlement|uezd") → Confession[]
export const MANUAL_OVERRIDES: Record<string, Confession[]> = {
  // "Гореловка|Ахалкалакский уезд": ["molokan"],
};
```

Применяется в `categorizeParish` поверх эвристики — для случаев, когда автоматика ошибается. Подсказки для заполнения берутся из `confession-audit.json`.

## 6. UI/UX изменения

- Легенда «Конфессия / община» на основной карте получает 12 строк (вместо 8). На десктопе колонка слева остаётся одной — высота помещается. На мобайле остаётся в свёрнутом виде по умолчанию (как уже сделано).
- В карточке точки добавляется строка «Конфессия» — список выявленных категорий (значки кружков с тултипом полного названия), чтобы пользователь видел, почему точка попадает под несколько фильтров.
- Tooltip над пунктом легенды показывает полное название (`confessions`), кнопка — короткое (`confessionsShort`).
- Мультиселект через Shift+click и «Показать всё / Очистить» — без изменений.

## 7. Технические детали

- `src/lib/parishCategory.ts` превращается в тонкий ре-экспорт: `CATEGORY_ORDER = CONFESSION_ORDER`, `CATEGORY_COLORS = CONFESSION_COLORS`, тип `ParishCategory = Confession`. Сигнатура `categorizeParish(props): Confession[]` (возвращает массив, не строку).
- Все места, где сейчас используется `category` (≈ строки 312, 326, 982, 1018 в `MapView.tsx`), переключаются на `categories`. Состояние фильтра остаётся `Set<Confession>`; начальное значение — все 12.
- Цвет круга на карте остаётся управляться `BUCKET_COLORS` (период книг). Категория — только видимость.
- Перевод/локализация: в `i18n.ts` удаляется блок `categoryName`; компонент тянет подписи напрямую из `tT(lang).confessionsShort`.
- Аудит-скрипт работает офлайн (Node/Bun), сетевых запросов нет.

## Что НЕ входит

- Перерисовка карточки церкви на карте Тбилиси (она уже использует эти же категории).
- Изменение самих данных `parishes.geojson` — категории вычисляются на лету, источник данных остаётся неизменным. Если потребуется «закрепить» категории в датасете — отдельной задачей.
- ML-классификация: используется только regex + таблицы ареалов (быстро, детерминированно, объяснимо).

## Порядок реализации

1. Перенос таксономии: `parishCategory.ts` → реэкспорт `Confession`. Обновить `i18n.ts` (удалить `categoryName`) и места его использования.
2. Новый `src/lib/confessionRules.ts` с `LOCATION_HINTS` и `KEYWORD_RULES`. Функция `categorizeParish(props): Confession[]`.
3. Обновить `MapView.tsx`: `categories` вместо `category`, новый фильтр-выражение, строка «Конфессия» в карточке точки.
4. Скрипт `scripts/audit-confessions.ts` + первый прогон → коммит `confession-audit.json` и `confessionOverrides.generated.ts`.
5. Пробежка по результату аудита, ручные правки для топ-проблемных кейсов (если есть), повторный прогон.
