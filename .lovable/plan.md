## Цель

Не выкидывать из приложения 2578 записей без координат. Сделать к ним удобный доступ через отдельную панель-справочник, которая открывается по кнопке и не мешает просмотру карты — особенно на мобильном.

## UX-решение

**Кнопка-триггер** в нижнем-левом углу карты (свободная зона, не конфликтует с зумом справа и языком сверху):

```
[ 📋 Без координат · 2578 ]
```

- Десктоп: компактная пилюля с иконкой и счётчиком.
- Мобильный: круглая FAB-иконка (📋) со счётчиком-бейджем, без текста.

**Открытие — адаптивный контейнер:**
- Десктоп (≥640px): выезжающая боковая панель справа (Sheet, ширина ~420px). Карта остаётся видимой и интерактивной слева.
- Мобильный (<640px): bottom-sheet на ~75% высоты экрана с drag-handle, чтобы свернуть свайпом вниз. Карту видно сверху.

Используем существующий `Sheet` из shadcn/ui (он уже работает и для side, и для bottom вариантов).

## Содержимое панели

1. **Заголовок**: «Селения без координат» / "Settlements without coordinates" + общий счётчик.
2. **Поле поиска** (sticky сверху): фильтрация по селению, церкви, региону, уезду — без учёта регистра, по подстроке. Дебаунс 150 мс.
3. **Фильтр по уезду** (компактный select/chips, опционально): быстрый отбор по району.
4. **Список** (виртуализированный — 2578 строк требуют windowing). Каждый item:
   - Селение (жирным) · Уезд, Регион
   - Церковь (если есть)
   - Годы службы
5. **Группировка** по уезду с заголовками-разделителями (sticky внутри списка) — помогает ориентироваться, когда поиск пуст.
6. **Действие на item**: «Найти на карте похожее» — если в датасете с координатами есть запись с тем же селением+уездом, центрируем карту на ней и закрываем панель. Иначе — кнопка неактивна с подсказкой «координаты неизвестны».

## Технические детали

**Данные.** Сейчас `build-geojson.ts` молча отбрасывает строки без координат (строка 100: `if (!isFinite) continue`). Нужно их сохранять во второй артефакт:

- Новый файл `public/data/unlocated.json` — массив объектов `{settlement, church, region, uezd, years, startYear}` с парами `{en, ru}` как в GeoJSON.
- Дедупликация по ключу `settlementRu|uezdRu` с объединением церквей и годов (как уже делается для координатных групп).
- Stats: добавить `unlocatedGroups` рядом с `withoutCoords`.

**Изменения в `scripts/build-geojson.ts`**:
- Не делать `continue` при отсутствии координат — складывать в отдельный массив `unlocatedRaw`.
- Группировать по `settlementRu+uezdRu`, формировать `unlocated.json`.
- Записать рядом со `stats.json`.

**Новый компонент `src/components/map/UnlocatedPanel.tsx`**:
- Загружает `/data/unlocated.json` лениво, при первом открытии (не блокирует загрузку карты).
- Внутренний поиск + фильтр по уезду в локальном state.
- Виртуализация: используем `@tanstack/react-virtual` (уже типичен для TanStack-стека; добавим, если ещё нет).
- Адаптив: проверяем `matchMedia("(min-width: 640px)")` и выбираем `side="right"` или `side="bottom"` для `Sheet`.

**Кнопка-триггер**: рендерим в `MapView.tsx` абсолютным позиционированием (`absolute bottom-4 left-4 z-10`). Не трогаем native MapLibre controls.

**i18n**: добавить ключи в `src/lib/i18n.ts` — `unlocated.button`, `unlocated.title`, `unlocated.search`, `unlocated.filterUezd`, `unlocated.empty`, `unlocated.findOnMap`, `unlocated.noCoords`.

**«Найти на карте похожее»**: при открытии панели строим Map<key, featureId> по существующему GeoJSON для O(1) lookup.

## Что меняется

- `scripts/build-geojson.ts` — сохраняем записи без координат в отдельный артефакт.
- `public/data/unlocated.json` — новый артефакт (генерируется скриптом).
- `public/data/stats.json` — добавляется `unlocatedGroups`.
- `src/components/map/UnlocatedPanel.tsx` — новый компонент.
- `src/components/map/MapView.tsx` — встраиваем кнопку и панель.
- `src/lib/i18n.ts` — новые строки.
- `package.json` — добавляется `@tanstack/react-virtual`, если ещё не установлен.

## Что НЕ делаем

- Не показываем эти точки на карте (нет координат — нечего показывать; иначе это создаст ложное впечатление).
- Не трогаем алгоритм геокодирования и существующую очистку координат.
- Не меняем поведение существующего поиска по карте — это отдельная панель.
