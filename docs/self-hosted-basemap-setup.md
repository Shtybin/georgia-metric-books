# Подложка карты: Stadia Maps (Alidade Smooth)

После долгих попыток с самохостингом Protomaps на R2 (см. историю в git)
мы перешли на готовый хостинг **Stadia Maps**. Это убирает всю возню
с pmtiles, rclone, CORS и 373-мегабайтным файлом.

**Что используем:**
- Стиль: `alidade_smooth` — визуально близок к CARTO Positron / Light.
- Источник: `https://tiles.stadiamaps.com/styles/alidade_smooth.json`
- Тарифы: free tier — **200 000 тайл-запросов в месяц** + 2 500 статичных
  карт. Для нашего трафика — с большим запасом.
- На `localhost` и `*.lovable.app` работает **без ключа** (они в whitelist
  Stadia для разработки). Ключ нужен только для нашего кастомного
  домена `metrics.datatells.info`.

---

## Шаг 1. Создать аккаунт Stadia Maps

1. Перейти на <https://client.stadiamaps.com/signup/>.
2. Зарегистрироваться (email + пароль). Платёжная карта **не нужна**
   для free tier.
3. После подтверждения email — вход в [client.stadiamaps.com](https://client.stadiamaps.com).

## Шаг 2. Создать Property для кастомного домена

В Stadia **одна Property = один домен** в Authentication Configuration
(несколько доменов в одну Property добавить нельзя). Поэтому делаем
одну Property под наш прод-домен — этого достаточно.

1. Дашборд → **Properties** → **Create a new property**.
2. Имя: `metrics.datatells.info`.
3. Открыть Property → секция **Authentication Configuration** →
   подсекция **Domains** → добавить **ровно один** домен:
   `metrics.datatells.info`.
4. Сохранить.

> **Почему не добавляем `georgia-metric-books.lovable.app`?**
> Все `*.lovable.app` (и `localhost:*`) уже в dev-whitelist Stadia —
> тайлы там грузятся без ключа. Превью в Lovable будет работать
> само собой. Отдельная Property под `lovable.app` не нужна.
>
> Если в будущем появится **ещё один** собственный домен — заводим
> под него **новую** Property и **новый** ключ (в один билд можно
> положить только один ключ; см. ниже).

## Шаг 3. Сгенерировать API-ключ

1. В Property `metrics.datatells.info` → секция **API Keys** →
   **Create API key**.
2. Скопировать значение (показывается один раз; если потерял —
   удалить и создать новый).

## Шаг 4. Добавить ключ в проект через Lovable Cloud

В Lovable **нет** отдельной вкладки «Environment Variables» в Project
Settings — переменные хранятся как **секреты Cloud**. Для нашего
`VITE_STADIA_API_KEY` это делается так:

1. В чате попроси меня добавить секрет (или я сам вызову нужный
   инструмент) — откроется защищённая форма «Add secret».
2. Имя секрета: **`VITE_STADIA_API_KEY`** (именно с префиксом `VITE_`,
   иначе Vite не вошьёт его в клиентский бандл).
3. Значение: ключ из Шага 3.
4. После сохранения Lovable автоматически пересобирает проект —
   ключ окажется в бандле и на превью, и на прод-домене.

> Префикс `VITE_` обязателен: эти переменные **попадают в публичный
> клиентский бандл** (это нормально для Stadia — ключ всё равно
> защищён whitelist-ом домена). Никаких серверных секретов с
> префиксом `VITE_` хранить нельзя.

**Локальная разработка:** если запускаешь `bun run dev` у себя на
машине, ключ можно добавить строкой в локальный `.env`
(`VITE_STADIA_API_KEY=...`). Но на `localhost` он не обязателен —
работает по dev-whitelist Stadia.

## Шаг 5. Проверить

Открыть карту в браузере:
- На `https://metrics.datatells.info/map` — тайлы летят на
  `tiles.stadiamaps.com/...?api_key=...` со статусом 200.
- На превью `*.lovable.app` — те же тайлы, без `api_key` в URL
  (или с ним — оба варианта Stadia принимает на dev-доменах).

Если на проде получаешь **401/403** — либо в Property не тот домен,
либо ключ не подхватился (проверь, что секрет назван **точно**
`VITE_STADIA_API_KEY` и проект был пересобран после добавления).

## Тарифы и мониторинг

- Free tier: 200 000 запросов тайлов / мес. Один пользователь смотрит
  карту ≈ 30–80 тайлов за сеанс → запас на 2 500–6 500 уникальных
  визитов в месяц.
- Использование видно в дашборде Stadia: **Properties → Usage**.
- Если упрёмся в лимит — апгрейд на `$20/мес` (1 млн запросов)
  или возвращаемся к самохостингу через R2 (старая инструкция
  сохранена в git history).

## Атрибуция

Стиль `alidade_smooth.json` уже содержит атрибуцию
(`© Stadia Maps © OpenMapTiles © OpenStreetMap`) в `sources[*].attribution`.
MapLibre рендерит её в правом нижнем углу автоматически — ничего
дописывать в коде не нужно.

## Troubleshooting

**Карта белая / серая, в консоли 401/403 от `tiles.stadiamaps.com`**
→ Домен в Property не совпадает с реальным (проверь точное написание,
без `https://` и без слэша в конце) **или** секрет
`VITE_STADIA_API_KEY` не задан / задан с опечаткой в имени.

**Работает локально, не работает на проде**
→ На `metrics.datatells.info` ОБЯЗАТЕЛЕН ключ + домен в Authentication
Configuration. На `localhost` и `*.lovable.app` ключ необязателен.

**Карта работает, но в Stadia дашборде нулевой usage**
→ Hard reload (Cmd/Ctrl+Shift+R) — старые тайлы могут отдаваться из
кэша браузера, не доходя до Stadia.

**Хочется тёмную карту**
→ В `src/lib/map-style.ts` поменять `alidade_smooth` на
`alidade_smooth_dark` (тот же free tier, тот же ключ, та же Property).

**Появился второй собственный домен**
→ Stadia не даёт класть два домена в одну Property. Варианты:
(а) завести вторую Property + второй ключ и держать **два разных
билда** (по одному ключу на билд); (б) поднять на нашей стороне
прокси, который подставляет нужный ключ по `Referer`; (в) апгрейд
тарифа Stadia — на платных планах лимит на домены/Property выше.
Для текущего проекта это пока не актуально.
